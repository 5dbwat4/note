# OOP 22-23 Mock 1

## 判断题

<PTAQuiz true-or-false correct={true} title="1-1">
  <Q>
    对单目运算符重载为友元函数时，可以说明一个形参。而重载为成员函数时，不能显式说明形参。
  </Q>

  <Explanation>
    单目运算符作为成员函数重载时，左操作数就是当前对象 `*this`，因此不显式写参数；作为友元或普通非成员函数时，需要一个形参表示操作数。
  </Explanation>
</PTAQuiz>

<PTAQuiz true-or-false correct={true} title="1-2">
  <Q>
    虚函数是用`virtual` 关键字说明的成员函数。
  </Q>

  <Explanation>
    虚函数是在类中用 `virtual` 声明的成员函数，用于通过基类指针或引用实现运行时多态。
  </Explanation>
</PTAQuiz>

<PTAQuiz true-or-false correct={true} title="1-3">
  <Q>
    使用提取符(&lt;&lt;)可以输出各种基本数据类型的变量的值，也可以输出指针值。
  </Q>

  <Explanation>
    题干把 `<<` 称为“提取符”不严谨，`<<` 通常是插入运算符；但它确实可以输出基本类型值和指针值。
  </Explanation>
</PTAQuiz>

<PTAQuiz true-or-false correct={true} title="1-4">
  <Q>
    因为静态成员函数不能是虚函数，所以它们不能实现多态。
  </Q>

  <Explanation>
    静态成员函数没有 `this` 指针，不能是虚函数；C++ 的动态多态依赖虚函数机制，因此静态成员函数本身不能实现运行时多态。
  </Explanation>
</PTAQuiz>

<PTAQuiz true-or-false correct={false} title="1-5">
  <Q>
    重载`operator+`时，返回值的类型应当与形参类型一致。

    比如以下程序中，`operator+`的返回值类型有错：

    ```cpp
    class A {
        int x;
    public:
        A(int t=0):x(t){ }
        int operator+(const A& a1){ return x+a1.x; }
    };
    ```
  </Q>

  <Explanation>
    `operator+` 的返回类型不必须和形参类型一致。示例中返回 `int` 表示把两个 `A` 对象内部的 `x` 相加，这在语法上是允许的。
  </Explanation>
</PTAQuiz>

<PTAQuiz multiple-choice correct={2} title="2-5">
  <Q>
    关于虚函数的描述中，（ ）是正确的。
  </Q>

  <C>
     虚函数是一个`static` 类型的成员函数
  </C>

  <C>
    虚函数是一个非成员函数
  </C>

  <C>
    基类中说明了虚函数后，派生类中与其对应的函数可不必说明为虚函数
  </C>

  <C>

  </Explanation>
</PTAQuiz>

<PTAQuiz multiple-choice correct={2} title="2-7">
  <Q>
    在下面类声明中，关于生成对象不正确的是（ ）。

    ```cpp
    class point
    { public:
    int x;
    int y;
    point(int a,int b) {x=a;y=b;}
    };
    ```
  </Q>

  <C>`point p(10,2);`</C>

  <C>`point *p=new point(1,2);`</C>

  </Explanation>
</PTAQuiz>

<PTAQuiz multiple-choice correct={2} title="2-8">
