## 浙大26春OOP L1 Personal Diary

### Personal Dariy

The Personal Dairy is a CLI (Command Line Interface) software.
It stores dairies in a plain text file like this:
```
2021-01-01:Happy New Year
2021-01-02:Happy New Year 2
```
For the test, the original file should be like this.
Each line is one entry for one day. The date is in "yyyy-mm-dd" format and followed by a comma ":". There is no duplicated lines for the same date.
The software consists of four programs:
```
pdadd 
pdlist
pdshow 
pdremove 
```
`pdadd` is used to add an entity into the diary for the date. It takes the date from the command line argument and reads the entry from the standard input. If an entity of the same date is in the diary, the existing one will be replaced. 
`pdlist` lists all entities in the diary ordered by date. This program lists to the standard output.
`pdshow` prints the content of the entity specified by the date to the standard output.  It takes the date from the command line argument. When there is no such entry at the date, it prints: `yyyy-mm-dd:No entry found for the given date`.
`pdremove` removes one entity of the date. It returns 0 at success and -1 at failure. It takes the date from the command line argument. When there is no such entry at the date, it prints: `Entry not found at yyyy-mm-dd`.
The following script shows how to use the software: (`>` is the command line prompt.)
```
>./pdadd 2024-02-29
started
>./pdlist
2021-01-01:Happy New Year
2021-01-02:Happy New Year 2
2024-02-29:started
>./pdshow 2021-01-01
2021-01-01:Happy New Year
>./pdshow 2022-02-01
2022-01-01:No entry found for the given date
>./pdremove 2021-01-02
>echo $?
0
>./pdremove 2022-01-02
Entry not found at 2022-01-02
>echo $?
255
>./pdlist
2021-01-01:Happy New Year
2024-02-29:ended
```
A `Makefile` is provided to make the software. Modify the objects definition in the Makefile for your code.

## 浙大26春OOP L2 Score Processing

### Score Processing

Write a program to process students score data.

The input of your program has lines of text, in one of the two formats:

1. Student's name and student id, as `<student id>, <name>`, and
2. Score for one student of one course, as `<student id>, <course name>, <marks>`.

Example of the two formats are:

```
3190101234, Zhang San
3190101111, Linear Algebra, 89.5
```

Comma is used as the seperator of each field, and will never be in any of the fields. Notice that there are more than one word for name of the person and name of the course. To make your code easier, the score can be treated as `double`.

The number of the students and the number of the courses are not known at the beginning. The number of lines are not known at the beginning either. The lines of different format appear in no order. One student may not get enrolled in every course.

Your program should read every line in and print out a table of summary in .csv format.

The first line of the output is the table head, consists fields like this:

```
student id, name, <course name 1>, <course name 2>, ..., average
```

where the course names are all the courses read, in alphabet order. There should be one space after each comma.

Then each line of the output is data for one student, in the ascended order of their student id, with score of each course, like:

```
3190101234, Zhang San, 85.0, , 89.5, , , 87.3
```

For the course that hasn't been enrolled, leave a blank before the comma, and should not get included in the average. The average has one decimal place. There should be one space after each comma.

And the last line of the output is a summary line for average score of every course, like:

```
, , 76.2, 87.4, , , 76.8
```

All the number output, including the averages have one decimal place.

### Input Format

As described in the text above.

### Output Format

As described in the text above.
The standard output is generated by a program compiled by gcc, that the round of the first decimal place is in the "gcc way".

## 浙大26春OOP L4 DoME

### DoME

The acronym DoME stands for Database of Multimedia Entertainment.

In essence, DoME is an application that lets us store information about CDs and DVDs. The idea is to create a catalogue of all the CDs and DVDs I own (or all I have ever seen or listened to).

The functionality that we want to provide with DoME should at least include the following:
- It should allow us to enter information about CDs and DVDs.
- It should store this information permanently so that it can be used later.
- It should allow us to print lists, such as a list of all DVDs in the database, or a list of all CDs.

The details we want to store for each CD are:
- the title of the album;
- the artist (name of the band or singer);
- the number of tracks on the CD;
- the total playing time;
- a ‘got it’ flag that indicates whether I own a copy of this CD; and s a comment (some arbitrary text).

The details we want to store for each DVD are:
- the title of the DVD;
- the name of the director;
- the playing time (we define this as the playing time of the main feature); s a ‘got it’ flag that indicates whether I own a copy of this DVD; and
- a comment (some arbitrary text).

Once we have defined the CD and DVD classes, we can create as many CD and DVD objects as we need – one object per CD or DVD that we want to store. Apart from this, we then need another object: a database object that can hold a collection of CDs and a collection of DVDs.

A `main()` is provided in the main.cpp. It creates four items and adds them into the database. After that, it reads lines from the standard input. The lines are in a comma seperated format, followed by the type name, `CD:` or `DVD:`. For CD, the contents are `Artist,Number of tracks,Playing time, Got it, Comment`. For DVD, they are `Director, Title, Playing time, Got it, Comment`.

Below are two example lines:
```
CD: John Marker, 7,74,1,nothing
DVD: Wala Wala, Go East, 74,1,fine movie
```
A line as `END` is used to indicat the end of the input. `0.in` file is the sample input.
The `main()` lists all the items entered. Each item is printed like a ticket, surrounded by two lines of 40-dash:
```
----------------------------------------
----------------------------------------
```
For CD, the ticket looks like:
```
----------------------------------------
Artist: John Marker
Number of tracks: 7
Playing time: 74 minutes
Got it: yes
Comment: nothing
----------------------------------------
```
For DVD:
```
----------------------------------------
Director: Wala Wala
Title: Go East
Playing time: 74 minutes
Got it: yes
Comment: fine movie
----------------------------------------
```
Since the `main()` is provided and not changeable, be careful the functions used in the `main()`.

```cpp
class Fraction{
private:
    long long fz;
    long long fm;
    friend istream& operator>>(istream& is, Fraction& f);
    friend ostream& operator<<(ostream& os, const Fraction& f);
    long long gcd(long long a, long long b){return b==0 ? a : gcd(b, a%b);}
    long long abs(long long a){return a<0?-a:a;}
    void opt(){
        if(fm<0){
            fz=-fz;
            fm=-fm;
        }
        long long g=gcd(abs(fz),fm);
        fz/=g;
        fm/=g;
    }
public:
    Fraction(){
        fz=0;
        fm=1;
    }
    Fraction(long long fenzi, long long fenmu): fz(fenzi),fm(fenmu){opt();}
    Fraction(double val){
        if(val==0){
            fz=0;
            fm=1;
            return;
        }
        fz = (long long)(val * 1e10 + (val >= 0 ? 0.5 : -0.5));
        fm = 1e10;
        opt();
    }
    operator double(){
        int x=fz/fm;
        int last = fz%fm;
        double xiaoshu=(double)(last)/fm;
        return (double)x+xiaoshu;
    }
    Fraction& operator+(Fraction& other){
        Fraction* res = new Fraction();
        res->fm = fm*other.fm;
        res->fz = fz*other.fm + fm*other.fz;
        res->opt();
        return *res;
    }
    Fraction& operator-(Fraction& other){
        Fraction* res = new Fraction();
        res->fm = fm*other.fm;
        res->fz = fz*other.fm - fm*other.fz;
        res->opt();
        return *res;
    }
    Fraction& operator*(Fraction& other){
        Fraction* res = new Fraction();
        res->fm = fm*other.fm;
        res->fz = fz*other.fz;
        res->opt();
        return *res;
    }
    Fraction& operator/(Fraction& other){
        Fraction* res = new Fraction();
        res->fm = fm*other.fz;
        res->fz = fz*other.fm;
        res->opt();
        return *res;
    }
};

istream& operator>>(istream& is, Fraction& f){
    char slash;
    is >> f.fz >> slash >> f.fm;
    return is;
}
ostream& operator<<(ostream& os, const Fraction& f){
    os << f.fz << '/' << f.fm;
    return os;
}
```

```cpp
template <class T>
class Vector{
private:
    T* data;
    size_t size;
    size_t capacity;

    void expand(){
        size_t newc = (capacity == 0) ? 1 : capacity*2;
        T* new_data = new T[newc];
        for(size_t i=0;i<size;i++){
            new_data[i] = data[i];
        }
        delete[] data;
        data = new_data;
        capacity = newc;
    }
public:
    Vector(): data(nullptr), size(0), capacity(0){}
    Vector(Vector& other){
        size=other.size;
        capacity=other.capacity;
        data = new T[capacity];
        for(size_t i=0;i<size;i++){
            data[i] = other[i];
        }
    }
    size_t add(T x){
        if(size==capacity) expand();
        data[size++] = x;
        return size-1;
    }
    void remove(int index){
        if(index >= size) return;
        for(size_t i=index;i<size-1;i++){
            data[i]=data[i+1];
        }
        size--;
    }
    int get_size(){
        return size;
    }
    const T& operator[](size_t index){
        return data[index];
    }
};
```

## 浙大26春OOP PI ADIF处理软件

### ADIF数据处理

# ADIF数据处理



### 背景：ADIF

ADIF是一种数据文件格式，以文本文件构成，文件内包含一个说明性的头和记录。
`<eoh>`表示头的结束。头的内容不是数据的一部分，可以被处理程序忽略。
由`<eor>`表示一条记录的结束，每条记录可以有多个字段，每个字段有不一致的长度，不同的记录可以有不同的字段。
每一个字段由形如`<A:n>`的标记开头，表示接下去的`n`个字符是`A`这个字段的内容，`A`是字段名。
如：

```adif
WriteLog
<EOH>
<QSO_DATE:8>20070526 <TIME_ON:6>010631 <FREQ:6>21.013 <BAND:3>15m <STX:1>1 <MODE:2>CW 
    <CALL:8>JH1GUO/4 
    <RST_SENT:3>599 
    <RST_RCVD:3>599 
    <SRX:3>024 
    <APP_WRITELOG_P:1>1 
    <APP_WRITELOG_MULS:5>    1 
    <PFX:3>JH4 
    <APP_WRITELOG_COUNTRY:5>Japan 
    <APP_WRITELOG_PREF:2>JA 
<EOR>
<QSO_DATE:8>20070526 <TIME_ON:6>010813 <FREQ:6>21.013 <BAND:3>15m <STX:1>2 <MODE:2>CW 
    <CALL:6>JA1MCU 
    <RST_SENT:3>599 
    <RST_RCVD:3>599 
    <SRX:3>022 
    <APP_WRITELOG_P:1>1 
    <APP_WRITELOG_MULS:5>    2 
    <PFX:3>JA1 
    <APP_WRITELOG_COUNTRY:5>Japan 
    <APP_WRITELOG_PREF:2>JA 
<EOR>
```

一条记录可以正好在一行内，也可以不在一行内。



### 要求：

编写一个ADIF文件处理程序，读入多个ADIF文本文件的数据，输出一个csv格式汇总表，汇总所有读入记录的字段信息。



### 输入格式：

从标准输入读入多个ADIF文件的数据。



关于标签（`<A:n>`中的$A$或`<B>`中的$B$）：

- 一个标签始于第一个匹配的`<`，终于第一个匹配的`>`。因此，`<LA<BEL:1>`是一个合法的标签，而`<LA>BEL:1>`不是。
- 所有样例输入保证`<A:n>`中的$n$是不超过$100$的正整数，`<A:n>`中的$A$或`<B>`中的$B$不包含换行和空格等空字符。
- 形如`<A:n>`的标签均是有效的，称为字段名标签。
- 形如`<B>`的标签只有`<eor>`和`<eoh>`是有效的。
- 无效的标签需要被忽略。
- 不同的字段标签可能有大小写的不同，需要忽略大小写的不同。
- 不同的记录中可能有不同的字段，但是输出时，需要把所有的字段名改成大写。



关于字段（出现在形如`<A:n>`的字段名标签之后）和记录（以`<eor>`标记结束）：

- 一个字段出现在形如`<A:n>`的字段名标签之后，那么它的长度应为恰好$n$个字符。如果在$n$个字符之后仍有内容，这部分内容应被忽略。例如：`<LABEL:3>ABCD`代表`LABEL`字段内容为`ABC`，而非`ABCD`。
- 字段中可能出现空字符和**汉字**，但不会出现换行。在ADIF文件格式中，每个**汉字**视为$2$个字符。
- 一条记录仅当以`<eor>`标记结束才是完整有效的。一个ADIF文件仅在末尾处可能出现无效的记录。无效的记录需要被忽略，参考输入样例3。
- (`QSO_DATE`, `TIME_ON`)的组合是记录的主键，每条完整有效的记录均有`QSO_DATE`和`TIME_ON`字段，且同一个样例里每条记录的主键唯一，不与其他记录相同。



关于ADIF文件格式：

- 多个ADIF是首尾相接的，参考输入样例2。
- 每个ADIF文件只有**至多一个**头，且一定出现在文件的最前面。
- 头中可能出现空字符，但不含符号`<`，否则将违背字段标签的第一条规则。
- ADIF文件可能是空的，或没有任何有效记录。



题目保证每个样例中：

- 标签长度不超过$100$个字符；
- 每个字段长度不超过$1000$个字符；
- 每个头长度不超过$10000$个字符；
- 每条记录中字段数量不超过$100$；
- 记录数与头数之和不超过$10000$；
- 总长不超过$10^6$个字符。



### 输出格式：

以csv格式输出对输入的ADIF文件的字段和记录内容的统计。

- 无论一个样例中输入了几个ADIF文件，均放在同一张表格中一起处理，参考输出样例2。
- 所产生的csv文件中，不包含ADIF文件的头的内容。
- 所有的字段按照字母顺序排列，所有的记录按照(`QSO_DATE`, `TIME_ON`)的组合排序，升序输出。
- 输出的csv表格的第一行中所有出现过字段名都应出现，且均为全部大写。
- 如果某条记录里面没有出现其他记录出现过的字段名，那么在csv表格该记录对应的行中，该字段所在列为空串。

另外，对于csv格式有如下特别提示：

- 注意在csv文件中，如果某一格内容中含有逗号`,`，需要在内容前后加双引号，如`"Kim, John"`。

## 浙大26春OOP PII Game of Castle

### Game of Castle

### Story

Adventure is a CLI game. The player has to explore in the castle with many levels and a lot of rooms. The task of the player is to find a room where the princess is prinsoned and take her leave the castle. There are many types of rooms, and each type of room has different types of exits. Note that there's a monster in one of the rooms, which the exact location is not able to be aware. But once the player meets a monster, the game is over.

When the game starts, the player is at the lobby of the castle. Then the program shows information about the lobby's name of the room, how many exits are there, and names of all exits (e.g.: "east", "south", "up"), like:

```
> Welcome to the lobby. There are 3 exits as: east, west and up.
> 
> Enter your command:
```


The player then can input "go" followed by the name of one exit to enter the room connected with that door, like:

```
> go east     
```


The player goes into the room to the east. The program shows the information about that room, like what happened in the lobby just now. And the player may input command to choose another room.\
Once the player enters a room with a monster, the program shows a message and game over. Once the player enters the room of princess, the program shows a message about the princess, and the process is going to leave with the player. The player then has to find their way out the castle. The only way to leave the castle is via the lobby.

All printed messages and user input are in English to simplify the code.

### Requirement

* At least three different kinds of room;
* At least five rooms;
* The room with monster or princess is randomly set;
* Put all source code files in one directory;
* Submit a zip of all the source code files, without any executable files.
