<PTAQuiz multiple-choice title="不是面向对象程序设计的主要特征" correct={3} meta={{label: "2-1", score: 2, difficulty: 0, author: "余春艳", org: "福州大学"}}>
  <Q>（&emsp;&emsp;&emsp;）不是面向对象程序设计的主要特征。</Q>
  <C>封装</C>
  <C>继承</C>
  <C>多态</C>
  <C>结构</C>
  <Explanation>
    **正确答案是 D（结构）。**

    面向对象程序设计的三大主要特征是**封装**、**继承**和**多态**。"结构"是结构化程序设计的概念，属于面向过程编程范式，不是面向对象程序设计的主要特征。

    - **A. 封装**：是OOP基本特征，将数据和操作数据的方法绑定在一起（如C++中的类）。
    - **B. 继承**：是OOP基本特征，允许派生类继承基类的属性和方法，实现代码复用。
    - **C. 多态**：是OOP基本特征，允许不同对象对同一消息作出不同响应（如虚函数、函数重载）。
    - **D. 结构**：是C语言中`struct`的概念，属于面向过程/结构化程序设计，不属于OOP三大特征。
  </Explanation>

<PTAQuiz multiple-choice title="关于cin和cout的错误说法" correct={2} meta={{label: "2-3", score: 2, difficulty: 0, author: "李廷元", org: "中国民用航空飞行学院"}}>
  <Q>下列关于 `cin` 和 `cout` 的说法中，错误的是____。</Q>
  <C>`cin` 用于读入用户输入的数据</C>
  <C>`cout` 用于输出数据</C>
  <C>`cin` 比C语言中的 `scanf()` 函数更有优势，它可以读取空格</C>
  <C>`cout` 通常与 `<<` 运算符结合</C>
  <Explanation>
    **正确答案是 C（`cin` 比C语言中的 `scanf()` 函数更有优势，它可以读取空格）。**

    题干要求选出错误说法。`cin` 使用 `>>` 运算符读取数据时，默认以空白字符（空格、制表符、换行符）作为分隔符，会**跳过**这些空白字符，因此它**不能**读取空格。要读取包含空格的字符串，需要使用 `getline()` 或 `cin.getline()`。

    - **A. `cin` 用于读入用户输入的数据**：正确。`cin` 是C++中输入流对象，用于从标准输入读取数据。
    - **B. `cout` 用于输出数据**：正确。`cout` 是C++中输出流对象，用于向标准输出写入数据。
    - **C. `cin` 比C语言中的 `scanf()` 函数更有优势，它可以读取空格**：错误。如前所述，`cin >>` 默认跳过空白，不能直接读取空格字符。
    - **D. `cout` 通常与 `<<` 运算符结合**：正确。`<<` 是流插入运算符，用于将数据输出到 `cout`。
  </Explanation>
</PTAQuiz>

<PTAQuiz multiple-choice title="不能连续输出3个值的语句" correct={1} meta={{label: "2-4", score: 2, difficulty: 0, author: "董俊", org: "燕山大学"}}>
  <Q>下列语句中，不能连续输出3个值的是。</Q>
  <C>`cout << x << y << z;`</C>
  <C>`cout << x, y, z;`</C>
  <C>`cout << x; cout << y; cout << z;`</C>
  <C>`cout << (x,y,z) << (x,y,z) << (x,y,z);`</C>
  <Explanation>
    **正确答案是 B（`cout << x, y, z;`）。**

    题干要求选出**不能**连续输出3个值的语句。选项B中使用了逗号运算符，但这行代码先执行 `cout << x`（输出 `x` 的值），然后依次计算表达式 `y` 和 `z`（但不输出它们的结果），因此只有 `x` 被输出，不能输出三个值。

    - **A. `cout << x << y << z;`**：可以。利用 `<<` 运算符连续串联，依次输出`x`、`y`、`z`三个值。
    - **B. `cout << x, y, z;`**：不能。逗号运算符的优先级低于 `<<`，只有 `x` 被输出，`y` 和 `z` 只被求值但不输出。
    - **C. `cout << x; cout << y; cout << z;`**：可以。三条独立语句，分别输出`x`、`y`、`z`。
    - **D. `cout << (x,y,z) << (x,y,z) << (x,y,z);`**：可以输出三个值（虽然都是`z`的值）。`(x,y,z)` 是逗号表达式，值为 `z`，所以三次都输出 `z` 的值，确实连续输出了3个值。
  </Explanation>
</PTAQuiz>

<PTAQuiz multiple-choice title="cin是什么" correct={3} meta={{label: "2-5", score: 2, difficulty: 0, author: "张丽娜", org: "沈阳师范大学"}}>
  <Q>在C++中，`cin` 是（）。</Q>
  <C>预定义的类</C>
  <C>预定义的函数</C>
  <C>一个标准的语句</C>
  <C>预定义的对象</C>
  <Explanation>
    **正确答案是 D（预定义的对象）。**

    `cin` 是C++标准库中 `istream` 类的一个**预定义（全局）对象**，在 `<iostream>` 头文件中声明，代表标准输入流。它不是类、函数或语句。

    - **A. 预定义的类**：错误。`cin` 属于 `istream` 类，但 `cin` 本身是该类的一个对象（实例），不是类。
    - **B. 预定义的函数**：错误。`cin` 是对象，虽然可以通过成员函数（如 `cin.get()`）进行操作，但 `cin` 本身不是函数。
    - **C. 一个标准的语句**：错误。`cin` 不是C++语句（如 `if`、`for`、`return` 等），而是流对象。
    - **D. 预定义的对象**：正确。`cin` 是 `istream` 类的全局对象，`cout` 和 `cerr` 同理分别是 `ostream` 类的全局对象。
  </Explanation>
</PTAQuiz>

    - **A. `iostream.h`**：错误。`iostream` 提供基础的输入输出流对象（`cin`、`cout` 等），但不包含 `setw` 等格式控制算子。

    - **C. `iomanip.h`**：正确。`iomanip`（I/O manipulators）头文件包含了 `setw`、`setprecision`、`setfill` 等流操纵算子。
    - **D. `stdlib.h`**：错误。`stdlib` 是C标准库头文件，包含 `malloc`、`free`、`atoi` 等函数，与流格式控制无关。
  </Explanation>

<PTAQuiz multiple-choice title="getline与cin混用" correct={1} meta={{label: "2-7", score: 2, difficulty: 0, author: "黄龙军", org: "绍兴大学"}}>
  <Q>
    有代码如下：
    ```cpp
    int n;
    string s;
    cin >> n;
    getline(cin, s);
    cout << s.size() << endl;
    ```
    则在输入以下数据后得到结果是（&emsp;&emsp;&emsp;）。
    ```plain
    1
    Hello World
    ```
  </Q>
  <C>11</C>
  <C>0</C>
  <C>5</C>
  <C>12</C>
  <Explanation>
    **正确答案是 B（0）。**

    这是 `cin >>` 与 `getline()` 混用时的经典问题。当输入为：
    ```
    1\n
    Hello World\n
    ```
    时，`cin >> n` 读取整数 `1`，但**换行符 `\n` 仍残留在输入缓冲区中**。紧接着 `getline(cin, s)` 会立即读到这个残留的换行符，导致 `s` 被赋值为空字符串 `""`，`s.size()` 返回 `0`。

    - **A. 11**：错误。11 是 "Hello World"（含空格）的字符数，但如果 `getline` 正常工作并读完第二行后确实应为11。但由于换行符残留问题，`getline` 实际读到的不是 "Hello World"。
    - **B. 0**：正确。见上文分析，`s` 为空字符串。
    - **C. 5**：错误。5 是 "Hello" 的长度，但此处不是用 `cin >>` 读字符串。
    - **D. 12**：错误。没有任何合理的计算会得到12。

    **解决方法**：在 `getline` 前使用 `cin.ignore()` 清除缓冲区中的换行符。
  </Explanation>
</PTAQuiz>

<PTAQuiz multiple-choice title="面向对象程序设计主要考虑" correct={3} meta={{label: "2-9", score: 2, difficulty: 0, author: "李祥", org: "湖北经济学院"}}>
  <Q>面向对象程序设计主要考虑的是提高软件的 ▁▁▁▁▁。</Q>
  <C>可靠性</C>
  <C>可移植性</C>
  <C>可修改性</C>
  <C>可重用性</C>
  <Explanation>
    **正确答案是 D（可重用性）。**

    面向对象程序设计的核心目标是通过**封装**、**继承**和**多态**三大特性来提高软件的**可重用性**（Reusability）。继承使得派生类可以复用基类的代码；多态使得接口可以被多种实现共享；封装将数据和行为绑定，形成独立可复用的模块。

    - **A. 可靠性**：虽然OOP有助于提高可靠性（通过封装隐藏实现细节、减少耦合），但这不是OOP最主要的设计目标。结构化编程也可以很可靠。
    - **B. 可移植性**：可移植性主要取决于是否使用标准库、不依赖平台特性等，与编程范式关系不大。
    - **C. 可修改性**：封装使修改内部实现不影响外部接口，但这更多是OOP的副作用而非首要目标。
    - **D. 可重用性**：正确。OOP设计的首要目标是代码复用，`class` 的封装、继承和多态都是围绕"一次编写、多次使用"的理念。
  </Explanation>
</PTAQuiz>
