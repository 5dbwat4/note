<PTAQuiz true-or-false title="If you are not interested in the contents of an exception object" correct={true} meta={{label: "1-1", score: 1, difficulty: 0, author: "张德慧", org: "西安邮电大学"}}>
  <Q>
    If you are not interested in the contents of an exception object, the `catch` block parameter may be omitted.。
  </Q>
  <Explanation>
    正确答案：**TRUE**（命题成立）。

    `catch` 块的参数与函数参数类似：如果你需要访问异常对象的内容，就写出参数名，如 `catch (const MyExc& e)`，然后在块内通过 `e` 使用它；如果你**只关心异常的类型而不关心其内容**，则可以省略参数名，只写类型，例如 `catch (const MyExc&)` 甚至 `catch (int)`。这是合法的语法，编译器允许只声明类型而不命名形参。

    因此“若对异常对象内容不感兴趣，`catch` 块的参数可以省略（不写形参名）”这一说法是正确的。注意：能省略的是**参数名**，类型一般仍需保留（唯一可完全省略类型的是捕获一切的 `catch (...)`）。
  </Explanation>
</PTAQuiz>

  <Q>
    `catch (type p)` acts very much like a parameter in a function. Once the exception is caught, you can access the thrown value from this parameter in the body of a `catch` block.。

    例如 `throw 5;` 被 `catch (int e)` 捕获后，`e` 的值就是 5。因此该说法正确。（实践中推荐用 `const&` 接收以避免不必要的拷贝和对象切片，但这并不改变“可通过该形参访问抛出值”这一结论。）
  </Explanation>

<PTAQuiz true-or-false title="异常处理的catch{ }语句块必须紧跟try{ }语句块之后，这两个语句之间不能插入另外语句。" correct={true} meta={{label: "1-3", score: 1, difficulty: 0, author: "黄万丽", org: "曲阜师范大学"}}>
  <Q>
    {`
    异常处理的catch{ }语句块必须紧跟try{ }语句块之后，这两个语句之间不能插入另外语句。
    `}
  </Q>
  <Explanation>
    正确答案：**TRUE**（命题成立）。

    在 C++ 的异常处理结构中，`try` 块与其后的 `catch` 块在语法上是一个**整体**：`catch` 处理器必须**紧接**在 `try` 块（或上一个 `catch` 块）之后，中间不允许插入任何其它语句、声明或表达式。例如下面写法是非法的：

    ```cpp
    try { /* ... */ }
    int x = 0;          // 错误：try 和 catch 之间不能插入语句
    catch (int e) { /* ... */ }
    ```

    一个 `try` 可以跟随**多个** `catch`，但这些 `catch` 之间同样必须连续排列。因此“catch 必须紧跟 try、之间不能插入其它语句”的说法正确。
  </Explanation>
</PTAQuiz>

    第2行：	try{
    第3行：	   if(a==1) throw(a);

    第5行：	}
    第6行：	catch(int b){
    第7行：	   cout << “error! a = ” << b << endl;

      {

        cout << number1 << " / " << number2 << " is "
