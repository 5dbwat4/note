<PTAQuiz multiple-choice title="访问对象函数成员的格式" correct={1} meta={{label: "2-1", score: 2, difficulty: 0, author: "陈晓梅", org: "广东外语外贸大学"}}>
  <Q>假定`AA`为一个类，`a()` 为该类公有的函数成员，`x`为该类的一个对象，则访问`x`对象中函数成员 `a()` 的格式为（）</Q>
  <C>`x.a`</C>
  <C>`x.a()`</C>
  <C>`x->a()`</C>
  <C>`(*x).a()`</C>
  <Explanation>对象通过 `.` 运算符访问成员函数，且函数调用需要加括号 `()`。**正确答案是 `x.a()`**。`x.a` 缺少函数调用括号，不合法的调用语法。`x->a()` 和 `(*x).a()` 是**指针**访问成员的方式 —— 需要 `x` 为指针（如 `AA* x;`），但题目明确 `x` 是"该类的**一个对象**"（非指针），所以不适用。</Explanation>
</PTAQuiz>

<PTAQuiz multiple-choice title="类定义的正确说法" correct={0} meta={{label: "2-2", score: 2, difficulty: 0, author: "李廷元", org: "中国民用航空飞行学院"}}>
  <Q>下列关于类定义的说法中，正确的是</Q>
  <C>类定义中包括数据成员和函数成员的声明</C>
  <C>类成员的缺省访问权限是保护的</C>
  <C>数据成员必须被声明为私有的</C>
  <C>成员函数只能在类体外进行定义</C>
  <Explanation>**正确答案：类定义中包括数据成员和函数成员的声明**。这是类的标准组成。其余错误原因：C++ 中 `class` 的**缺省访问权限是 `private`**（私有的），不是 `protected`（保护的）；数据成员**不必须**声明为私有，可根据需要设为 `public` 或 `protected`；成员函数**既可以在类体内直接定义**（内联），也可以在类体外通过 `::` 定义。</Explanation>
</PTAQuiz>

<PTAQuiz multiple-choice title="类和对象的错误叙述" correct={0} meta={{label: "2-3", score: 2, difficulty: 0, author: "李廷元", org: "中国民用航空飞行学院"}}>
  <Q>下列关于类和对象的叙述中，错误的是</Q>
  <C>一个类只能有一个对象</C>
  <C>对象是类的具体实例</C>
  <C>类是对某一类对象的抽象</C>
  <C>类和对象的关系是一种数据类型与变量的关系</C>
  <Explanation>题目问的是**错误**的叙述，**正确答案：一个类只能有一个对象**（这是错误的）。一个类可以创建**任意多个**对象（实例），如同 `int a, b, c;` 可以定义多个 `int` 变量。其余均为正确叙述：对象是类的具体实例（实例化）；类是对某一类对象的抽象描述；类与对象的关系类似于数据类型（如 `int`）与变量（如 `int x`）的关系 —— 类是自定义类型，对象是该类型的变量。</Explanation>
</PTAQuiz>

<PTAQuiz multiple-choice title="::的作用域解析符" correct={3} meta={{label: "2-4", score: 2, difficulty: 0, author: "翁恺", org: "浙江大学"}}>
  <Q>Resolver `::` is used to:</Q>
  <C>Define a member function outside class declaration</C>
  <C>Access a member of a namespace</C>
  <C>Access a static member of a class</C>
  <C>All of the others</C>
  <Explanation>**正确答案是 "All of the others"**。`::` 作用域解析符（scope resolution operator）在 C++ 中有三种常见用途：① 在类声明**外部定义成员函数**，如 `void X::f() { }`；② 访问**命名空间**中的成员，如 `std::cout`；③ 访问类的**静态成员**，如 `ClassName::staticVar`。以上全都是 `::` 的合法用法。</Explanation>
</PTAQuiz>

<PTAQuiz multiple-choice title="类的实例化" correct={1} meta={{label: "2-5", score: 2, difficulty: 0, author: "陈晓梅", org: "广东外语外贸大学"}}>
  <Q>类的实例化是指（&emsp;&emsp;）。</Q>
  <C>定义类</C>
  <C>定义对象</C>
  <C>调用类的成员函数</C>
  <C>访问对象的数据成员</C>
  <Explanation>**正确答案：定义对象**。类的**实例化**（instantiation）指根据类这个"蓝图"创建出一个具体的对象，如 `ClassName obj;`。"定义类"是编写类的声明/定义，不是实例化；"调用成员函数"和"访问数据成员"是在对象创建之后才能进行的操作，它们不是实例化本身。</Explanation>
</PTAQuiz>

<PTAQuiz multiple-choice title="函数声明不需要什么" correct={3} meta={{label: "2-6", score: 2, difficulty: 0, author: "范鹏程", org: "内蒙古师范大学"}}>
  <Q>C++函数的声明和定义可以分开。函数声明不需要(&emsp;&emsp;&emsp;&emsp;)。</Q>
  <C>返回类型</C>
  <C>函数名</C>
  <C>参数表</C>
  <C>函数体</C>
  <Explanation>**正确答案：函数体**。函数声明（函数原型）只需要告知编译器函数的**返回类型**、**函数名**和**参数表**（参数类型），例如 `int add(int a, int b);`。函数体 `{ ... }` 是函数定义时才需要的部分，声明中可以省略。返回类型、函数名和参数表在函数声明中都是必不可少的。</Explanation>
</PTAQuiz>

<PTAQuiz multiple-choice title="面向对象首先识别什么" correct={1} meta={{label: "2-7", score: 2, difficulty: 0, author: "范鹏程", org: "内蒙古师范大学"}}>
  <Q>在面向对象的程序设计中，首先需要在问题域中识别出若干个（&emsp;）</Q>
  <C>函数</C>
  <C>类</C>
  <C>文件</C>
  <C>过程</C>
  <Explanation>**正确答案：类**。面向对象程序设计（OOP）的核心思想是将现实世界中的事物抽象为**类**（`class`）。在分析设计阶段，首先要在问题域中识别出若干个类，再定义其数据成员（属性）和函数成员（行为），最后通过对象间的消息传递完成交互。函数、文件、过程都不是面向对象分析的首要识别对象 —— 类是 OOP 的基本单元。</Explanation>
</PTAQuiz>
