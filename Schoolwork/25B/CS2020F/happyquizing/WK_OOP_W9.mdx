<PTAQuiz true-or-false title="虚函数是用virtual 关键字说明的成员函数。" correct={true} meta={{label: "1-1", score: 1, difficulty: 0, author: "何振峰", org: "福州大学"}}>
  <Q>
    虚函数是用`virtual` 关键字说明的成员函数。
  </Q>
  <Explanation>
    **正确答案：正确（TRUE）**

    这是虚函数的**基本定义**。在 C++ 中，虚函数必须使用 `virtual` 关键字声明：
    ```cpp
    class Base {
    public:
        virtual void func();  // func 是虚函数
    };
    ```

    `virtual` 关键字告诉编译器为该函数建立虚函数表（`vtable`）条目，使得通过基类指针/引用调用时能够进行**动态绑定**（运行时根据实际对象类型选择正确的函数版本）。这是 C++ 多态的实现基础，该说法完全正确。
  </Explanation>
</PTAQuiz>

<PTAQuiz true-or-false title="将构造函数说明为纯虚函数是没有意义的。" correct={true} meta={{label: "1-2", score: 1, difficulty: 0, author: "何振峰", org: "福州大学"}}>
  <Q>
    将构造函数说明为纯虚函数是没有意义的。
  </Q>
  <Explanation>
    **正确答案：正确（TRUE）**

    构造函数**不能**被声明为虚函数或纯虚函数，原因如下：
    1. **构造函数的作用**是初始化对象，在构造过程中虚函数表指针（`vptr`）逐步建立，动态绑定机制此时尚未就绪，虚函数调用无法正常工作。
    2. **纯虚函数**意味着该类是抽象类，不能实例化——但构造函数的存在意义恰好是为了创建对象，两者矛盾。
    3. C++ 标准禁止构造函数使用 `virtual` 关键字，编译器会直接报错。

    因此试图将构造函数声明为纯虚函数既不合理也无意义，该说法正确。
  </Explanation>
</PTAQuiz>

<PTAQuiz true-or-false title="动态绑定是在运行时选定调用的成员函数的。" correct={true} meta={{label: "1-4", score: 1, difficulty: 0, author: "何振峰", org: "福州大学"}}>
  <Q>
    动态绑定是在运行时选定调用的成员函数的。
  </Q>
  <Explanation>
    **正确答案：正确（TRUE）**

    **动态绑定（Dynamic Binding）** 是 C++ 多态的实现机制：
    - 通过基类指针或引用调用虚函数时，调用的具体函数版本**在运行时**根据指针/引用所指向的**实际对象类型**确定，而非编译时确定。
    - 这一过程依赖于**虚函数表（`vtable`）** 和**虚函数表指针（`vptr`）**：在运行时通过 `vptr` 查找 `vtable` 中的函数地址。

    与之相对的是**静态绑定**（编译时确定）。该说法准确描述了动态绑定的含义，正确。
  </Explanation>
</PTAQuiz>

    - **B（保护成员）**：基类中的保护成员在私有继承下变为派生类的**私有成员**，派生类可以访问。
    - **C（公有成员）**：基类中的公有成员在私有继承下也变为派生类的**私有成员**，派生类可以访问。
    - **D（新增私有成员）**：派生类自己新增的私有成员当然是派生类的私有成员。

    本质区别：基类私有成员虽然内存在，但访问权限不属于派生类，因此**不可能**成为派生类的私有成员。
  </Explanation>
