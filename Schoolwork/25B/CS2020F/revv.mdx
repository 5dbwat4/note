# 概述

C++ 面向对象程序设计的特性主要包括以下四个方面
- 抽象（Abstraction）：抽象是面向对象方法的核心特性，它涉及到对具体问题或对象进行概括，抽出一类对象的公共性质并加以描述的过程。在C++中，抽象可以通过定义类来实现，类中包含数据成员（属性）和成员函数（行为）。数据成员描述了对象的静态特征，而成员函数则定义了对象可以执行的操作。
-封装（Encapsulation）：封装是将抽象得到的数据和行为相结合，形成一个有机整体的过程。在C++中，这通常通过类来实现，类将数据和操作数据的函数代码进行有机的整合，形成一个独立的实体。封装有助于隐藏对象的内部细节，只通过明确的接口与外部交互，从而提高了代码的安全性和可维护性。
- 继承（ Inheritance ）：继承是面向对象编程中的一个重要概念，它允许创建一个新类（派生类）继承自一个现有类（基类）。派生类继承了基类的所有属性和方法，并可以添加新的属性和方法或重写现有方法。通过继承，可以实现代码的重用和扩展，提高了软件开发的效率。
- 多态（ Polymorphism ）：多态是对人类思维方式的一种直接模拟，它允许同一个操作作用于不同的对象时，产生不同的结果。在 C++ 中，多态可以通过虚函数和动态绑定来实现。当派生类重写了基类的虚函数时，根据对象的实际类型来调用相应的函数实现，这就是多态的体现。多态提高了代码的灵活性和可扩展性。

# 类与对象

程序设计语言中的抽象主要分为：过程抽象和数据抽象

- 面向对象程序设计语言通过类（ class ）来实现封装，也可以说封装好之后的抽象数据类型称为类。
- 类具有封装能力，能够将抽象类型的数据和操作函数包装成一个整体，并将数据的内部结构和接口的实现细节隐藏起来，只向外界提供接口。除了能够通过接口访问类的功能之外，外部对象对类的内部构造和实现细节是一无所知的。

访问权限控制标识符  
• public  
– 可以从结构（类）的外部（使用者）访问  
– 实现抽象定义类的接口  
• private  
– 仅供结构（类）的内部（自身成员函数）访问  
– 实现信息隐藏  
• protected  
– 供结构（类）的内部及后代访问

在没有指定成员的访问权限时， struct 中的成员具有 public 权限，而 class 中的成员则具有 private 权限。

2. 底层剖析

**现象归类**：浅拷贝导致的指针悬挂（Dangling Pointer）与双重释放（Double Free）。

**核心原因**：
1. **默认赋值运算符/复制构造函数的行为**：由于 `String` 类未显式重载赋值运算符和复制构造函数，编译器生成默认的逐成员拷贝（浅拷贝）。其本质操作是将 `ptr` 指针的地址值直接复制，导致 `p1.ptr` 和 `p2.ptr` 指向堆区的**同一块内存单元**。
2. **资源生命周期的错配**：局部对象 `p2` 离开 `{ }` 作用域时，调用 `~String()` 执行 `delete ptr`，释放了该共享内存。
3. **指针悬挂与崩溃**：`p1` 的 `ptr` 指针因此成为**悬空指针**（指向已被操作系统回收或标记为无效的地址）。`p1.print()` 输出未定义行为（乱码/崩溃）。主函数结束，`p1` 析构函数再次执行 `delete ptr`，导致对同一内存区域进行两次释放，引发 **Double Free** 致命错误（程序崩溃）。
3. RAII 与 智能指针介绍

RAII（资源获取即初始化）：RAII 是 C++ 的核心设计哲学。其核心思想是将动态申请的资源（如堆内存、文件句柄）的生命周期绑定在栈对象的生命周期上。在对象的构造函数中获取资源，在析构函数中释放资源。当对象离开作用域（栈展开）时，析构函数自动被调用，从而保证资源的自动释放，避免资源泄漏。

C++ 标准库智能指针（基于 C++11/14/17）：标准库提供了一系列模板类，封装了裸指针并实现了 RAII 机制。

1. **`std::unique_ptr`（独占所有权）**
   * **特性**：智能指针独占托管对象的所有权。不支持拷贝语义（拷贝构造/赋值被 `= delete`），只支持移动语义（转移所有权）。
   * **底层开销**：内存占用与裸指针相同，无额外性能损耗。
   * **用途**：明确独有资源的所有者，适用于工厂模式、单一线程内的资源管理。

2. **`std::shared_ptr`（共享所有权）**
   * **特性**：采用引用计数机制管理对象生命周期。允许浅拷贝，每增加一个 `shared_ptr` 指向同一资源，引用计数（`use_count`）原子递增。当引用计数归零时，底层资源被自动释放。
   * **底层开销**：包含一个控制块（存储引用计数、自定义删除器等信息）。拷贝构造、析构函数涉及原子操作的引用计数增减，存在一定性能损耗。
   * **内存优化**：推荐使用 `std::make_shared` 构造。该函数将“数据内存”与“控制块内存”合并为一次连续堆内存分配，提升内存分配效率和 CPU 缓存命中率。

3. **`std::weak_ptr`（弱引用/观测者）**
   * **特性**：配合 `std::shared_ptr` 使用。不增加资源引用计数，主要用于观测 `shared_ptr` 所管理的对象。
   * **解决的问题**：解决 `shared_ptr` 之间产生的**循环引用（死锁）**问题。循环引用会导致双方引用计数无法归零，造成内存泄漏。
   * **访问限制**：不能直接解引用，必须通过 `.lock()` 成员函数获取一个临时的 `std::shared_ptr`。若原资源已释放，`.lock()` 返回空指针，从而保证绝对的安全访问。

对移动函数的几点说明

① **对象资源被移动之后，它应该是可析构和有效的**
- 对象的内存资源被移动后，并不会立即销毁对象，它仍处用可用状态，可以为它赋新值，因此指针被移动之后，应将它置为 `nullptr`（可析构）。

② **拷贝左值，移动右值**
- 一个类同时设置了拷贝构造函数、赋值运算符函数、移动拷贝构造函数和移动赋值函数时，编译器就会使用与普通函数相同的参数匹配规则进行函数调用。当需要左值时，执行拷贝构造函数或赋值运算符函数；当需要右值时，执行移动拷贝或移动赋值函数。

② **如果类没有移动函数，右值也被拷贝**
- 如果一个类定义了拷贝构造函数和赋值运算符函数，但没有移动构造函数和移动赋值运算符，就不能执行移动操作，即使调用 `move` 函数也只能执行对象拷贝操作。

## 3、友元类

*   一个类可以是另一个类的友元，友元类的所有成员函数都是另一个类的友元函数，能够直接访问另一个类的所有成员（包括 public、private 和 protected）。
*   **【例】** 通过友元类的成员函数直接访问对象的私有成员。

```cpp
//Eg.cpp
#include <iostream>
using namespace std;
class A{
private:
    int x,y;
public:
    A(int i,int j){x=i;y=j;}
    int getX(){return x;}
    int getY(){return y;}
    friend class B;   // 声明类 B 是类 A 的友元类
};

class B{
private:
    int z;
public:
    int add(A a){ return a.x+a.y+z; }  // A 类对象作参数
    int mul(A a){ return a.x*a.y*z; }  // A 类对象作参数
    B(int i=0){ z=i; }
};

void main(){
    A a(2,3);
    B b(4);
    cout<<b.add(a)<<endl;   // 输出 9
    cout<<b.mul(a)<<endl;   // 输出 24
}
```

5. 友元使用注意事项

- **①** 在类域中的函数原型前加上关键字 `friend`，就将该函数指定为该类的友元了。关键字 `friend` 用于声明友元，它只能出现在类的声明中。
- **②** 友元函数**并非**类的成员函数，所以它不受 `public`、`protected`、`private` 的限定，无论将它放在 `public` 区，或者 `protected` 区，还是 `private` 区，都是完全相同的。
- **③** 友元**不具逆向性和传递性**。即，若 A 是 B 的友元，并不表示 B 是 A 的友元（除非特别声明）；若 A 是 B 的友元，B 是 C 的友元，也不能代表 A 是 C 的友元（除非特别声明）。
- **④** 友元使编程更简洁，程序运行效率也更高，但它可以直接访问类的私有成员，破坏了类的封装性和信息隐藏。不建议多用友元。


# 多态（
**1. 多态的概念**
- 多态是面向对象程序设计语言的又一重要特征，指的是**不同对象接收到同一消息时会产生不同的行为**。
- 简单地说，**多态**就是在同一个类或继承体系结构的基类与派生类中，用**同名函数来实现各种不同的功能**。

**2. 多态与继承的关系**
- 继承所处理的是类与类之间的层次关系问题
- 而多态则是处理类的层次结构之间，以及同一个类内部**同名函数的关系问题**。但通常是指继承结构中基类和派生类之间通过**同名虚函数实现不同函数功能**的问题。


OOP中广义的多态通常有 3 种表现形式
① 重载多态：包括函数重载和运算符重载；
② 模板多态：通过一个模板生成不同的函数或类（第 7 章介绍）；
③ 继承多态：通过基类对象的指针（引用），调用不同派生类对象的重定义同名成员函数，表现出不同的行为。一般情况下，多态即指这种类型。


1、什么是虚函数
- 用 `virtual` 关键字修饰的成员函数，`Virtual` 关键字其实质是告知编译系统，被指定为 `virtual` 的函数采用动态联编的形式编译。
- 只有类的成员函数才能声明为虚函数，普通函数（不属于任何类）不能定义为虚函数。
- 多态类：拥有虚函数的类也称为多态类。

2、虚函数的定义形式
```cpp
class x{
    ......
    virtual 返回类型 f(参数表);
}
```

3、虚函数的执行机制
- 如果基类中的非静态成员函数被定义为虚函数，且派生类覆盖了基类的虚函数，当通过基类的指针或引用调用派生类对象中的虚函数时，编译器将执行动态绑定，**调用到该指针（或引用）实际所指对象所在类中的虚函数版本**。


**overwrite**和**final**










[ ] 只能被重载为类的非静态成员函数，不能被重载为友元和普通函数。

对定义重载函数的下列要求中，（ ）是错误的。


A.要求参数的个数不同
B.要求参数中至少有一个类型不同
C.要求函数的返回值不同
D.要求参数个数相同时，参数类型不同
