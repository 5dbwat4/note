## **Part 1: Cache Structure**

**1. What is the cache block size (in words)?**
* **Offset bits:** The offset uses bits 4–0, which is 5 bits.
* **Bytes per block:** $2^5 = 32$ bytes per block.
* **Words per block:** 8 words. 

**2. How many blocks does the cache have?**
* **Index bits:** The index uses bits 9–5, which is 5 bits.
* **Total blocks:** $2^5 =$ **32 blocks**.

**3. What is the ratio between total bits required for such a cache implementation over the data storage bits?**
* **Data bits per block:** $32 \text{ bytes} \times 8 \text{ bits/byte} = 256 \text{ bits}$.
* **Overhead bits per block:**
    * Tag bits: 54 bits (bits 63–10)
    * Valid bit: 1 bit (standard for direct-mapped caches)
* **Total bits per block:** $256 \text{ (data)} + 54 \text{ (tag)} + 1 \text{ (valid)} = 311 \text{ bits}$.
* **Ratio:** 
$$
\frac{\text{Total Bits}}{\text{Data Bits}} = \frac{311}{256} \approx 1.21
$$
