## 实验3 SQL数据完整性
### 实验目的
熟悉通过SQL实现数据完整性控制的方法

### 实验平台
数据库管理系统：MySQL、OpenGauss、PolarDB

### 实验内容和要求
1. 定义包含主键、外键、CHECK约束的数据表
2. 插入数据，验证主键对实体完整性的控制作用
3. 删除被引用表数据，验证外键ON DELETE子句对参照完整性的控制
4. 修改被引用表主键，验证外键ON UPDATE子句对参照完整性的控制
5. 插入/修改数据，验证CHECK子句对校验完整性的控制
6. 定义触发器，通过修改表数据验证触发器功能
7. 按要求完成实验报告

## Class Project - MiniSQL
### 任务目标
1. 设计并实现精简型单用户SQL引擎MiniSQL，支持字符界面输入SQL语句完成表、索引的增删及记录的增删查
2. 提升系统编程能力，加深对数据库管理系统实现技术的理解

### 系统需求
1. **数据类型**：支持integer、char(n)、float三种基本类型
2. **表定义**：单表最多32个属性，支持属性unique声明、单属性主键定义
3. **索引定义**：主属性自动建立B+树索引，unique属性可由用户手动创建/删除B+树索引
4. **数据操作**：支持多条件（AND连接）等值与区间查询、单条记录插入、单条/多条记录删除
