# PTAQuiz

`PTAQuiz` is an MDX component group for writing interactive quiz blocks inside
notes. The content inside question, choice, answer, and explanation blocks is
normal MDX, so Markdown syntax, lists, math, images, and code blocks are handled
by the existing MDX pipeline before the quiz component receives them.

The component is intentionally explicit. A quiz must declare exactly one quiz
category attribute, and the public attributes are limited to the documented API
below.

## Import

```js
import { PTAQuiz, Q, C, A, Blank, PTACodeContent, Explanation } from "@md-comp/ptaquiz";
```

## Props

`PTAQuiz` supports only these quiz category attributes:

- `true-or-false`
- `multiple-choice`
- `multiple-answer`
- `fill-in-the-blank`
- `fill-in-the-blank-for-programming`
- `multiple-file`
- `programming`
- `subjective`
- `code-completion`

It also supports these data attributes:

- `correct`: answer for choice-like types. It is 0-indexed. Use a number for
  `multiple-choice`, an array of numbers for `multiple-answer`, and a boolean
  for `true-or-false`.
- `title`: optional title shown in the quiz header.
- `meta`: optional object. Hovering over the quiz header shows one line per
  entry, formatted as `[key]: [value]`.

No other `PTAQuiz` attributes are part of the API.

## Child Blocks

- `Q`: question body.
- `C`: choice body for `multiple-choice` and `multiple-answer`.
- `A`: reference answer body for `multiple-file`, `programming`, `subjective`,
  and `code-completion`.
- `Blank`: fillable answer span for fill-in-the-blank types.
- `PTACodeContent`: highlighted code content with inline `Blank` support for
  `fill-in-the-blank-for-programming`.
- `Explanation`: optional explanation. For answer-checking types, it can be
  expanded after answering; wrong answers open it by default.

## Multiple Choice

`multiple-choice` is a single-answer selection. `correct` is a 0-indexed number.
Clicking an option checks the answer immediately.

````html
<PTAQuiz multiple-choice correct={1} title="Single answer">
  <Q>
    Which option supports full **MDX** content?

    ```cpp
    int answer = 2;
    ```
  </Q>

  <C>Plain text only</C>
  <C>Markdown and MDX-rendered nodes</C>

  <Explanation>
    MDX compiles the child content before `PTAQuiz` renders it.
  </Explanation>
</PTAQuiz>
````

## Multiple Answer

`multiple-answer` is a multi-answer selection. `correct` is an array of
0-indexed numbers. The user selects all applicable choices, then submits.

```html
<PTAQuiz multiple-answer correct={[0, 2]}>
  <Q>Select every true statement.</Q>
  <C>`Q`, `C`, and `Explanation` are structural children.</C>
  <C>The parent reparses Markdown text at runtime.</C>
  <C>The parent receives Vue VNodes generated by MDX.</C>
  <Explanation>Runtime Markdown parsing is unnecessary here.</Explanation>
</PTAQuiz>
```

## True Or False

`true-or-false` uses a boolean `correct` value.

```html
<PTAQuiz true-or-false correct={true}>
  <Q>Markdown inside the question block is compiled before `PTAQuiz` sees it.</Q>
  <Explanation>`PTAQuiz` receives rendered VNodes from MDX.</Explanation>
</PTAQuiz>
```

## Fill In The Blank

`fill-in-the-blank` renders normal prose with inline `Blank` components. The
answer is the text inside each `Blank`.

When `Answer peek` is off, users fill blanks and submit. If any blank is wrong,
the correct answer is shown near that blank and `Explanation` opens by default.

When `Answer peek` is on, there is no submit action. Blanks are read-only; users
click a blank to reveal the answer inside the blank in green.

```html
<PTAQuiz fill-in-the-blank>
  Something to fill is <Blank>This is answer to fill</Blank>.

  <Explanation>
    This explanation is available after a wrong submission or in answer-peek mode.
  </Explanation>
</PTAQuiz>
```

## Programming Fill In The Blank

`fill-in-the-blank-for-programming` reuses the fill-in-the-blank behavior.
Use `PTACodeContent` when code highlighting is needed. The plain text around
`Blank` should be written with MDX template-string expressions so whitespace,
newlines, and indentation are preserved.

````html
<PTAQuiz fill-in-the-blank-for-programming>
  <Q>
    Complete the code.

    <PTACodeContent lang="cpp">{`#include <iostream>
using namespace std;

int main() {
  int x = `}
<Blank>42</Blank>
{`;
  cout << x << endl;
  return 0;
}`}
    </PTACodeContent>
  </Q>
</PTAQuiz>
````

IMPORTANT：建议使用
```
<Blank>{`cout << x`}</Blank>
```
这种写法，将要填写的东西左右包裹上花括号和backtick来避免HTML转义

## Solution Reveal Types

`multiple-file`, `programming`, `subjective`, and `code-completion` share the
same behavior: there is no validation and no input field. The component renders
the question and provides a `Show answer` / `Hide answer` toggle for `A`.

````html
<PTAQuiz programming>
  <Q>
    Write a function that returns the smaller of two integers.
  </Q>

  <A>
    ```cpp
    int min2(int a, int b) {
      return a < b ? a : b;
    }
    ```
  </A>
</PTAQuiz>
````

<PTAQuiz subjective title="Subjective answer reveal">
  <Q>
    Explain why MDX child content should be compiled before the quiz component
    handles interaction.
  </Q>

  <A>
    It keeps content rendering and quiz interaction separate. The quiz component
    only needs to classify child blocks and manage state.
  </A>
</PTAQuiz>
